Introduction

A lightweight component library build for quickly developing cards in HTML and CSS. The aim for this projects is to generate cohesiveness and efficiency when developing card based components.

Quick Start

Make sure your development environment is up to date. Use an HTML5 doctype with a viewport meta tag for proper responsive behavior. If your using VS Code as a text editor this can be done automatically by pressing ! followed by tab

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<link rel="stylesheet href="cards.css">
</head>
<body>

<! -- place all cards inside of a div tag with a class name of grid -- >
<div class="grid">
 <! -- cards go here -- >
</div>

 </body>
</html>

download or clone the repository and link the style sheet in the head of your document.


Cards on github

Overview

Cards is built using a 12 column grid system and CSS grids. If you're not familiar with CSS grids don't worry too much because cards handle all of the responsive behavior for you. If you would like to override these widths simply give the targeted element a property of grid-column and a value of span followed by the number of columns you wish for the card to span. If you would like a better understanding of how grids work in CSS view the attached link.



.className {
grid-column: span 6;
}

Hero Cards

Hero cards are designed to be main focal points of your site or application. They will automatically span 100% of the parent container or 12 columns so there is no need to pass a span value in the class name. These cards are image driven with a limited amount of text. They are broken into 3 different cards. Ones that are aligned left, center, and right.

Article Cards

Article cards are designed to be an entryway for a larger portion of writing. Whether that be blog posts or news articles these cards could be used to display a small summary of a body of writing, a title of the article or blog post, and a supporting image to compliment the content. Some varieties of these cards were not designed to have a summary but rather a category or tagline above the title. Product Cards

The buttons come with a hover effect build in that translates the y-axis 5px up. These cards as well should be passed a span value in their class name to determine how many columns they cover in the grid.
